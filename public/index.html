REST API · v0.0.1

Pokédex.API

A NestJS REST API for Pokémon data. Full CRUD with pagination, flexible lookup by ID, number, or name, and a seed endpoint that populates the database directly from PokéAPI.

NestJS 11 MongoDB 7 Mongoose 9 TypeScript 5

Endpoints

Pokémon

POST

/api/v1/pokemon

Create a new Pokémon

GET

/api/v1/pokemon

List all Pokémon — supports limit and offset query params

GET

/api/v1/pokemon/:term

Find by Mongo ID, Pokémon number, or name

PATCH

/api/v1/pokemon/:term

Update by Mongo ID, Pokémon number, or name

DELETE

/api/v1/pokemon/:id

Delete by Mongo ID

Seed

POST

/api/v1/seed

Populate the database with 650 Pokémon from PokéAPI

Query parameters — GET /api/v1/pokemon

Parameter Type Default Description
limit number 10 Number of results per page
offset number 0 Number of results to skip

Quick start

1. Start the database
docker compose up -d
2. Start the server
pnpm start:dev
3. Seed the database
curl -X POST http://localhost:3000/api/v1/seed
Response — GET /api/v1/pokemon?limit=2
{ "total": 650, "limit": 2, "offset": 0, "data": [ { "name": "bulbasaur", "no": 1 }, { "name": "ivysaur", "no": 2 } ] }

Environment variables

Variable Required Default Description
MONGODB_URI required — MongoDB connection string
